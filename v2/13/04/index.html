GCSE Link: 4.10 - The Fetch-Execute Cycle

The Fetch-Execute Cycle (also called the Fetch-Decode-Execute Cycle) is the sequence of operations that the CPU uses to carry out instructions.

The cycle consists of three stages:


  • The Fetch stage
     
    1. The memory address of the next instruction is copied from the Program Counter (PC) into the Memory Address Register (MAR).
    2. This address is sent along the address bus to main memory, and the instruction at that memory location is sent back along the data bus into the Memory Buffer Register (MBR).
    3. The contents of the MBR are copied into the Current Instruction Register (CIR).
    4. The PC is incremented to point to the next instruction.
  • The Decode stage
     
    1. The Control Unit (CU) splits the instruction into its opcode and operand(s) where relevant, and the required operation is identified.
  • The Execute stage
     
    1. The CU sends control signals to other CPU components (e.g. the ALU).
    2. The instruction is carried out. Data may be read from or written to memory, and registers (including the PC and Status Register) may be updated.

An interrupt is a signal sent by a device or program that requires immediate attention from the CPU. It allows the currently running process to be suspended, and then resumed after the interrupt request has been dealt with.

At the end of each Fetch-Decode-Execute cycle, the CPU checks for interrupt requests. If it has received one, it saves the current state of the registers in memory (to allow the registers to be used to handle the interrupt) and then calls an Interrupt Service Routine (ISR) to handle the request. After it has finished, the registers are repopulated with the original data and the cycle restarts.


Give some examples of when interrupts would be required.

For example, an input from a peripheral (e.g. keypress or mouse move), a hardware fault such as a power failure, software interrupts from the Operating System, or exception interrupts (e.g. divide-by-zero).
