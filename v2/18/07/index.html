The client-server model is a structure for applications involving clients and servers.

Clients can request data from the server, which will send a response. They communicate with each other using agreed protocols such as HTTP and FTP.

If most of the processing is done by the server it is called thin-client computing, whereas if the client has a significant processing capacity it is called thick-client computing.

The WebSocket protocol is an application layer protocol which establishes a connection between a web browser and server.

This allows for a persistent connection, meaning that data can be sent quickly by either side at any time. This is commonly used in instant messaging and online gaming.


CRUD (create, retrieve, update, delete) describes the four basic operations of a database.

These can be mapped to the four HTTP methods, and also to four basic SQL commands:

Table 1 shows a mapping of CRUD to the HTTP methods and SQL commands.

Table 1

CRUD HTTP SQL Meaning
Create POST INSERT Add data to the server
Retrieve GET SELECT Fetch data from the server
Update PUT UPDATE Modify data on the server
Delete DELETE DELETE Remove data from the server

A Representational State Transfer Application Program Interface (REST API) performs the translation between HTTP methods and SQL commands, allowing a client to make requests to the server.

A JavaScript (JS) file referenced by a HTML file can be run in the browser. This can create a HTTP request to the web server via the REST API, which can then pass on the request to the database. The database's response is passed back to the JS file in the browser and displayed to the user.

There are two main file formats used to transmit the response of the database: JavaScript Object Notation (JSON) (which are essentially JS dictionaries) and Extensible Markup Language (XML) (which looks like HTML code). Below are examples of both:

Example 1 shows a database response in JSON format.

Example 1
{
  "movie": {
    "title": "Avengers: Endgame",
    "year": 2019,
    "runtime": 181,
    "genre": ["action", "adventure", "sci-fi"]
  }
}

Example 2 shows a database response in XML format.

Example 2
<movie>
  <title> Avengers: Endgame </title>
  <year> 2019 </year>
  <runtime> 181 </runtime>
  <genre> action </genre>
  <genre> adventure </genre>
  <genre> sci-fi </genre>
</movie>


What are the advantages of JSON over XML?

JSON can be easily evaluated using JavaScript (since it uses the same syntax as JS objects) and is therefore quicker to parse than XML. JSON is also easier for a human to read due to its compact key-value format. Furthermore, it is easier to create due to its simpler syntax. JSON can also represent arrays.
