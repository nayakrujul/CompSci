GCSE Link: 5.05 - Network Security

A firewall is a device that monitors network traffic, acting as a barrier between the network and the Internet.

Firewalls are usually built-in to routers, although they can also be separate devices. They work by inspecting the headers of packets that go into and out of the network, and blocking some packets based on pre-defined rules. This process is known as packet filtering. Some computers also have software firewalls built-in to the OS: these use the same packet filtering concepts, but only protect a single computer and not the whole network.

There are two types of packet filtering: static and dynamic. Static packet filtering uses a list of blocked IP addresses and TCP ports, called the access control list. Only the header is inspected, not the payload, so this method is less secure. Dynamic packet filtering (also called stateful inspection) inspects the payload of the packet to determine if the packet should be allowed to enter or leave the network.

A proxy server is a device that handles network communications on behalf of a device.

The proxy server sits between a device and the Internet, and makes all requests to external servers instead of that device. This provides anonymity to the device, as the requests will have the proxy server's IP address instead of the device's.


Encryption provides protection against data being read if it is stolen or intercepted.

Encryption algorithms convert the original plaintext data into the encrypted ciphertext data. A password (called a "key") is needed to decrypt the ciphertext back into the plaintext. When the key used to encrypt the plaintext is the same key needed to decrypt the ciphertext, it is called symmetric encryption. When they are different keys, it is called asymmetric encryption.

For symmetric encryption, a problem arises: how can the sender securely transmit the encryption key to the receiver without it being intercepted? The key would have to be encrypted, but then the same problem arises for the new encryption key. Instead, we use asymmetric encryption:

Every computer has a two different keys: one public key and one private key. The public key can be shared everywhere, but the private key must be kept a secret. The keys are related through a mathematical algorithm which means that a message encrypted by one key can only be decrypted by the other. When Person A wants to send a message to Person B, they simply need to encrypt the message using B's public key. Now, the message can only be decrypted using B's private key (which only Person B has access to) providing a secure channel of communication.

The downside is that asymmetric encryption is much slower than symmetric encryption, especially for longer messages. Therefore, asymmetric encryption is only used at the start of the communication to exchange the key that will be used for the remainder of the communication. After this, symmetric encryption can be used without a problem.

A digital signature is used to verify the identity of the sender.

Remember that the asymmetric encryption process works both ways: messages can also be encrypted using the private key and then decrypted using a public key. Consider this series of steps:

  1. Person A wants to securely send a message to Person B, but also prove that it was them that sent the message.
  2. First, A generates a hash (a one-way algorithm which turns a message into a series of bits of a certain length, usually 256 or 512) of the message, and encrypts it with their own private key.
  3. This encrypted hash is then appended to the (unencrypted) message, and the whole thing is then encrypted using B's public key.
  4. B receives the encrypted message and decrypts it using their private key. This gives the message followed by an encrypted hash of the message.
  5. B can decrypt the encrypted hash using A's public key. B can then generate the hash of the message themselves and compare it to the decrypted hash. If they match, the hash must have been encrypted using A's private key (which proves that person A sent the message).

A Certificate Authority is responsible for issuing digital certificates that guarantee the authenticity of a public key.

The digital certificate contains the public key, the information about who owns the key, and how long the certificate is valid for. It contains the digital signature of the Certificate Authority, so it can be trusted by all parties.


Which network threats do firewalls protect against?

Firewalls can prevent malware from entering a network by blocking packets from suspicious sources. They can also block attackers from gaining unauthorised access to the system, and prevent denial of service attacks by blocking excessive requests.
