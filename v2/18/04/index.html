The Internet is a global network of networks set up to allow computers to communicate with one another around the world.

The Internet is organised into tiers of networks: Tier 1 networks are the largest, usually operated by global telecommunications companies; Tier 2 and 3 networks are smaller, usually servicing a single country or region.

The Internet Protocol (IP) handles the transmission of packets between networks, as described on page 18.01.

Routers and gateways are used to forward packets to another network closer to the destination using routing and forwarding tables. Routers connect networks using the same protocols, while gateways provide an interface for networks using different protocols, translating data formats as required. This enables communication between networks that otherwise wouldn't have been possible.


A Uniform Resource Locator (URL) is the full address used to identify and access a specific resource on the Internet.

For example, the URL of this page is https://cs.rujulnayak.com/v2/18/01/index.html. It is split into multiple parts:

  • The protocol: in this case https, but could also be http, ftp, etc.
  • The host: the domain name (or IP address) of the server, in this case cs.rujulnayak.com
  • The path: in this case /v2/18/01/index.html
  • A query (optional): starts with a ?, for example ?theme=dark
  • A section (optional): starts with a #, for example #top

A domain name is a human-readable hierarchical label used to identify an area of the internet.

Every domain name is made up of labels separated by dots. A top-level domain (TLD) is the right-most label. It can either be a generic TLD such as .com, .org or .net, or a country code TLD (ccTLD) such as .uk, .fr or .de. The next label from the right is called the second-level domain (2LD), and then the next is called the third-level domain (3LD) and so on.

A fully qualified domain name (FQDN) is one that specifies all labels up to the host. For example, cs.rujulnayak.com and aqa.org.uk are both FQDNs, while com and org.uk are both not.

The Domain Name System (DNS) is used to translate human-readable domain names into IP addresses.

Every domain has one or more IP addresses assigned to it (rujulnayak.com has the IP addresses 185.199.108.153, 185.199.109.153, 185.199.110.153 and 185.199.111.153). DNS stores a table of all domains and their assigned IP addresses to allow for conversion.

The Internet Assigned Numbers Authority (IANA) coordinates the allocation of IP addresses. The regional internet registries (RIRs) are five organisations that allocate IP address blocks to their regions. Each TLD is operated by a domain registry, and domain names are sold to end-users by domain registrars.


Why are the internet registries needed?

Global cooperation is required to ensure that each assigned IP address is unique. Without this, the same addresses could be issued twice, causing routing failures. Also, a hierarchical model allows for easier global scalability.
