Big data is a catch-all term for data that doesn't fit the usual containers.

The three main features of big data are the three 'V's:

  • Volume - data that is too big to fit onto a single server
  • Velocity - data which is rapidly created, modified and processed (within milliseconds)
  • Variety - data in many different forms, from text to videos

The lack of structure makes big data difficult to store in conventional formats like relational databases, and also difficult to analyse. Also, since big data is too large to fit onto a single server, the processing must be distributed across multiple machines.

Functional programming languages are perfect for processing big data as they are stateless (functions have no side-effects), use immutable data structures (which cannot be changed), and support higher-order functions. This makes it easier to write correct, efficient code that can be distributed to run across multiple servers.

Since we can't represent big data in a relational database, we instead use the fact-based model, where each individual piece of information is stored as a single immutable (unchangeable) fact. Since facts can't be changed or deleted, they each have a timestamp: if two facts have contradictory information, the one with the more recent timestamp is used.

We can show the structure of these facts with graph schema, where circles represent entities, rectangles represent properties of those entities (joined to the circles using dashed lines), and solid lines represent relationships between the entities.
