A function is a rule that maps elements of one set (the domain) to elements of another (the co-domain).

Every element of the domain must have an associated output value in the co-domain, however not every element of the co-domain must be used, and multiple inputs may map to the same output. The notation f: A → B means that the function with identifier f has domain A and co-domain B. Passing in particular inputs to a function is called function application: the arguments are simply written after the function identifier, for example f "Hello".

For example, consider the function square: Int → Int. Given an integer as input, it will output the square of the input. Each element of the domain (the integers) is mapped to an element of the co-domain (also the integers), but not every element of the co-domain is used (for example, no input maps to 2). Also, multiple inputs can map to the same output (e.g. both 5 and -5 map to 25).


Some functions may have multiple arguments, in which case its domain is the Cartesian product of the types of its arguments. For example, if we have a function called mult which multiplies two integers together, its type is Int × Int → Int. Although we would say that it takes two arguments, the function can be thought of as taking a single argument which is a tuple of two real numbers, for example (3, 4).

In reality however, a function with multiple arguments is not actually evaluated all at once. Instead, the function is first partially applied to its first argument, which returns another function which is then applied to the second argument. For example, if we are calling mult 3 4, this is first partially applied to 3, which gives a new function mult3 that takes a single argument and multiplies it by 3. This new function is then applied to 4 which returns 12. Therefore, the type signature of the function can be viewed as Int → (Int → Int), i.e. a function which takes an integer and returns [a function which takes an integer and returns an integer]. We can drop the brackets to simply get Int → Int → Int.


Function composition means chaining two functions f and g together such that the output of f becomes the input of g. Note that the co-domain of f must be the same as the domain of g for this to work. For example, we could compose our mult function from before with a new isEven function with type Int → Bool. This works because the co-domain of mult and the domain of isEven are both Int. This composition creates a new function which we write as isEven∘mult (or in Haskell, isEven.mult).


A first-class object is one that can appear in expressions, be assigned to variables, be assigned as arguments, and be returned in function calls.

In functional programming languages, functions are first-class objects, just like integers, reals, strings and booleans.


In functional programming languages, lists are implemented as the concatenation of a head (the first element in the list) and a tail (the rest of the list). In Haskell, this is written as head:tail.

A higher-order function is one that takes a function as an argument, or returns a function as a result.

There are three main higher-order functions that you need to know:

  • map takes a function func and a list lst, and applies func to each item in lst. It returns a new list (of the same length as lst).
  • filter takes a function func and a list lst, and returns only the elements of lst for which func applied to that item is True.
  • fold (or reduce) takes a function func, a list lst, and a starting value val. It repeatedly applies the function to val and each item in lst, and then assigns the value back to val. The final result is a single value.

For example, folding a list on addition starting from 0 is the same as summing the list.


What is the type and signature of a function which takes the three Real side lengths of a cuboid and outputs the volume?

The type is Real × Real × Real → Real . The signature is Real → Real → Real → Real .
