2B.03 – Structured Programming

Computer Science Revision


Previous: 2B.02 - Local and Global Variables

Structured programming involves decomposing the program into manageable modules.

Simple subroutines can then be written to carry out individual tasks, and the main program can be written using these subroutines.


The advantages of structured programming are:

  • The program is easier to code (you only need to code the simple subroutines)
  • Testing individual tasks is easier (the tasks are already in subroutines)
  • Maintenance is easier (you don't need to look through the whole code to see what needs changing)
  • Subroutines can be used in other programs (they are now reusable)
